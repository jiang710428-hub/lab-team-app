# 项目记忆

## 项目概述
- **项目类型**：实验室/科研团队管理App（移动端，iOS风格）
- **角色**：作为产品经理，按用户要求进行App设计与页面还原
- **产出形式**：高保真HTML原型（单文件，内联CSS+JS+SVG），多页面间用相对路径跳转

## 重要约定（必须遵守）
- **所有页面默认加完整交互，不需要用户提醒**。每个页面所有可点击元素都必须绑定真实行为：
  - 返回箭头 → 跳转回上一页（`href` 或 `history.back()` 或 `goBack()`）
  - Tab → 切换高亮 + 内容切换/提示
  - 搜索 → 输入回车执行筛选
  - 筛选/更多/操作按钮 → 弹出底部 ActionSheet
  - 详情 → 弹出居中 Modal
  - 列表项/卡片 → 可点击查看详情
  - 悬浮按钮 → 弹出添加方式选择
  - 点击有 `:active` 缩放反馈 + `-webkit-tap-highlight-color: transparent`
- **返回按钮用 `<div onclick>` 而非 `<a onclick>`**（无 href 的 `<a>` 在部分浏览器不响应点击）
- 页面间跳转使用相对路径（如 `home.html`、`material.html`），文件放在同一目录
- 交互统一用：底部 ActionSheet（`.mask`+`.sheet`）、居中 Modal、Toast 提示
- **路由返回策略**：返回按钮统一用 `history.back()` 或直接 `location.href` 跳目标页，**不能混用**。material-detail 的 goBack() 用 history.back()，warehouse-detail 返回用 location.href='warehouse.html'
- 路由动态返回：`goBack()` 函数根据 URL `from` 参数决定返回目标
- **禁止使用 `confirm()`/`alert()`**：预览iframe环境中会被阻止，删除/确认操作一律用自定义弹窗（mask+居中卡片 或 mask+底部sheet）

## 出库录入要点
- **扫码出库(outbound-entry.html)**：物料/选择批次/领用人均无交互(纯展示)；选择批次和领用人保留右箭头图标；选择仓库有交互(底部弹窗)
- **物料出库(outbound-material.html)**：选择物料/选择批次/领用人均无交互但保留右箭头；选择仓库有交互(底部弹窗)；无扫码框；"+添加物料"按钮可动态新增卡片
- 物料卡合并了：表单字段 + 出库清单子区域 + 领用人 + 图片 + 周期性使用完(全部在一张白卡内)
- 备注独立卡片(不与物料同卡)
- 物料字段必须根据仓库筛选(演示固定为试剂+仓库"乃东路一号库")
- 出库清单是物料卡内的子区域，按"批次"分组显示条目
- 批次条目展示3列：库存 / 已选 / 单价
- 点击批次条目进入 outbound-batch.html 子页面进行勾选
- 子页面通过 localStorage 共享勾选状态
  - 扫码出库：STORAGE_KEY='outbound_entry_state'，结构 {selections, currentBatches}
  - 物料出库：STORAGE_KEY='outbound_material_state'，结构 {cards:[{selections,currentBatches,wh,periodic},...]}
  - 子页面通过 ?source=material&idx=N 参数区分读写哪个storage
